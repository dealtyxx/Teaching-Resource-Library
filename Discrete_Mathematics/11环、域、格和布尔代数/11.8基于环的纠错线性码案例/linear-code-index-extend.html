循环码 · 多项式环

循环码 · CRC（F₂[x]/(xⁿ−1)）

码字 = 信息 ⊕ 余数。CRC 把信息多项式 m(x)·xʳ 除以生成多项式 g(x)，余数即校验位。蓝=信息，绿=CRC，红=错误。

➗ CRC = 多项式环上的取模运算

把比特串看成 F₂ 上的多项式，CRC 校验码 = m(x)·xʳ mod g(x)（环 F₂[x] 中的取余）。接收端再除以 g(x)：余数为 0 则无错，非零则检出错误。循环码正是理想 ⟨g(x)⟩ 在商环 F₂[x]/(xⁿ−1) 中的体现——这是 CRC、BCH、RS 的统一代数框架。

码率与汉明界 · 冗余权衡

码率 R=k/n 越高越省带宽，纠错能力 t=⌊(d−1)/2⌋ 越强越费冗余。汉明界给出二者的理论上限。

⚖️ 鱼与熊掌：码率 vs 纠错

汉明界（球填充界）：2ᵏ·Σᵢ₌₀ᵗ C(n,i) ≤ 2ⁿ。重复码与汉明码取等号，是完备码（冗余用到极致）。提高纠错能力 t 必然牺牲码率 R——工程上需按信道质量在汉明界内权衡，这正是系统观念的体现。

RS 码 · 突发纠错与交织

RS 码在符号（GF(2ᵐ)）上纠错，一个符号错≈纠一串比特。配合交织，可把长突发打散到多个码字。

🛰️ 符号级纠错 + 交织 = 抗突发

RS(n,k) 是 MDS 码，d=n−k+1，可纠 t=⌊(n−k)/2⌋ 个符号错误。一次突发噪声只损坏少数符号，故 RS 天生抗突发；再用交织把相邻符号分散到不同码字，更能化整为零。CD/DVD、二维码、深空通信（如北斗、探测器）都靠它守护数据。
