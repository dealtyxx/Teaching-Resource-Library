RBAC + 拓扑排序

🔐 RBAC 角色继承（偏序的上闭包）

高级角色继承所有下级角色的权限。选一个角色，其有效权限 = 所有 ≤ 它的角色权限之并（传递闭包）。

🔐 有权必有责

角色继承用偏序的传递性表达：管理员 ≥ 编辑 ≥ 普通用户 ⟹ 管理员拥有它们的全部权限。权限随层级累积，也意味着责任随层级加重。

🗂️ 令行禁止、有序高效

拓扑排序保证：任何任务都在其全部前置完成后才执行。偏序约束下仍可能有多种合法顺序——这正是工程管理『统筹兼顾、有序推进』的数学保证。

📐 偏序关系三性质验证

以 RBAC 角色继承关系为例，逐条验证偏序（≤）必须满足的三条性质。

✓ 自反性 Reflexive
∀a: a ≤ a

每个角色都继承自身的全部权限——管理员 ≤ 管理员，恒成立。

✓ 反对称性 Antisymmetric
a ≤ b ∧ b ≤ a ⟹ a = b

角色继承图无环：若编辑 ≤ 管理员且管理员 ≤ 编辑，则两者必是同一角色——保证等级不会循环、权限边界清晰。

✓ 传递性 Transitive
a ≤ b ∧ b ≤ c ⟹ a ≤ c

普通用户 ≤ 编辑 ≤ 管理员 ⟹ 普通用户 ≤ 管理员。权限沿层级逐级累积，这正是 RBAC『角色继承』与拓扑排序得以成立的根基。

📐 三性质=偏序的根基

自反、反对称、传递缺一不可：少了反对称会出现『循环上级』，少了传递则继承与调度无法跨级推导。古代官制的森严等级，与现代系统的权限/调度，共享同一套偏序公理。
